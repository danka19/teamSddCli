# FAQ: как начать работу с teamSddCli

`teamSddCli` помогает команде вести изменения через OpenSpec, роли, локальные проверки и доказательства. Он не принимает решения за людей, не публикует во внешние системы и не заменяет инженерную ответственность.

Канонический источник: [описание проекта](../README.md) и [спецификация FAQ](../../openspec/changes/add-product-faq-and-role-runbook/specs/product-faq-and-role-runbook/spec.md).

Если команда ещё не подготовлена, владелец процесса начинает с `sdd setup <папка> --confirm`. Если ситуация уже известна, используйте `sdd start <ситуация>`; для существующего change — `sdd next --change <путь>`. Команда объяснит следующий безопасный шаг, но не изменит внешние системы.

## Поддержка FAQ

FAQ — поддерживаемый обзор, а не вторая политика. Владелец процесса проверяет его при принятии изменений, затрагивающих `sdd`, роли, статусы или границы AI; автор change обновляет страницу и запускает `python scripts/validate_product_faq.py --json`. Первый проход нового оператора записывается как синтетическая локальная проверка: без секретов, сети, внешних интеграций и заявлений о принятии решения.
