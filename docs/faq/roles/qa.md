# QA: проверить доказательства качества

Канонический источник: [artifact and lifecycle gates](../../runbooks/ARTIFACT_AND_LIFECYCLE_GATES.md).

Цель — проверить сценарии, регрессию и доказательства без фиктивного принятия. Начните с `sdd next --change <путь> --role QA`. Сохраняйте результаты проверок и найденные блокеры. QA не переводит lifecycle и не выпускает change: при конфликте критериев эскалируйте Tech Lead.
