# Дизайн самодостаточных локальных skills репозитория

## Цель

Положить регулярно используемые workflow-skills в репозиторий так, чтобы они
обнаруживались Codex из `.agents/skills/`, не зависели от
`~/.codex/skills`, не обращались во внешний интернет и не изменяли внешнее
состояние.

## Архитектура

- `process/agent-skills/` — канонический versioned источник skills.
- `.agents/skills/` — отслеживаемая Git проекция для Codex discovery.
- Имена skills используют префикс `teamssd-`, чтобы не конфликтовать с
  глобальными версиями.
- Все внутренние ссылки остаются repo-local.
- `process/agent-skills/README.md` кратко объясняет структуру, offline-границы
  и добавление адаптерных каталогов других runtime, например
  `.gigacode/skills/`.

## Состав

Пакет включает проектное управление, OpenSpec, планирование, выполнение,
локальное ревью, отладку, проверку и отчётность. Из исходных skills удаляются
или заменяются явной остановкой:

- web, browser search, MCP и connectors;
- внешние URL и API;
- `git push`, `pull`, `fetch`, `clone`, `gh` и автоматическое создание PR;
- установка пакетов и глобальные пути;
- внешняя публикация и deployment.

Локальные Git-операции, OpenSpec CLI, чтение и изменение файлов репозитория,
локальные команды проекта и создание коммитов разрешены.

## Распространение

Codex использует `.agents/skills/`. Другой runtime создаёт собственную
проекцию в отдельном каталоге, например `.gigacode/skills/`, не меняя
каноническую tool-agnostic логику. Новый адаптер добавляется через отдельное
согласованное изменение с описанием поддерживаемого формата и ограничений.

## Проверка и остаточный риск

По решению владельца автоматические и поведенческие тесты для этого изменения
не создаются. Перед завершением выполняется только ручная статическая проверка:
frontmatter, уникальность имён, repo-local ссылки, отсутствие внешних операций
и соответствие проекции каноническим файлам.

Без поведенческих тестов нельзя доказать, что все будущие агенты одинаково
интерпретируют инструкции. Offline-граница подтверждается содержимым файлов,
а не сетевой sandbox-политикой runtime.
