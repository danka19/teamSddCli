# Session Report

Write the report in Russian. For bounded work include task, changes, local
verification and next step. For phase, architecture, OpenSpec or multi-file
work also explain decisions, trade-offs, user impact, manual checks, residual
risks, documentation, skills, subagents and commit state.

Make the report self-contained; links support the explanation but do not
replace it. State waived or unavailable checks explicitly.

Do not push, create a PR, publish or deploy as part of reporting.
