# Requesting Code Review

Prepare a local review package containing requirements, accepted design,
relevant diff, verification evidence and known risks. Ask the reviewer for two
explicit verdicts: specification compliance and implementation quality.

Treat blocking findings as unresolved until fixed and re-reviewed. Preserve
human decisions and do not expand scope from reviewer suggestions without
change intake.

Review local files only. Do not use hosted review APIs or create a remote PR.
