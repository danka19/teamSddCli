# OpenSpec Apply Change

Select the change, read local status and apply instructions, then read every
reported context file. Confirm branch, roadmap ownership, task progress and
verification expectations.

Implement pending tasks in order, keep edits scoped, collect proportionate
local evidence and update checkboxes only after evidence exists. Use
`teamssd-test-driven-development` for code behavior unless the human explicitly
waives it.

Finish with docs sync, strict OpenSpec validation, roadmap validation and
`teamssd-session-report`. Stop before remote Git, PR creation or publication.
