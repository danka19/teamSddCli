# Writing Plans

Map affected files and interfaces first. Split work into the smallest coherent
tasks that have independent acceptance evidence. Each task names exact paths,
inputs, outputs, local verification commands, documentation updates and a
commit boundary.

Copy project-wide constraints into the plan and remove placeholders or vague
steps. Respect explicit human exceptions such as an approved no-test scope and
record their residual risk.

Save plans under `docs/superpowers/plans/`. Use only local evidence and tools.
