# Phase Change Intake

Restate the idea, classify it, compare it with the active phase and accepted
OpenSpec requirements, then choose exactly one route: adopt now, queue in the
phase, create an OpenSpec change, defer, or reject.

Record the decision in the smallest canonical project artifact. Product
behavior or contract changes require OpenSpec before implementation. Continue
the previous work unless safety, accepted requirements or the human requires a
stop.

Use local files only and never mutate external systems.
