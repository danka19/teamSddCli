# Phase Status Audit

Compare roadmap phase status, detailed phase plans, active OpenSpec lifecycle,
task checkboxes, human acceptance and audit evidence. Never infer acceptance
from completed tasks or passing checks.

Report unfinished, unmarked, contradictory or pending-acceptance items with
their exact source. Use `teamssd-roadmap-openspec-validator` when roadmap and
OpenSpec coexist. Correct only unambiguous documentation drift; human
acceptance and risk decisions remain human-owned.

Inspect local repository files only.
