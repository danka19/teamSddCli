# Design: аналитическое интервью в packaged companion

## Контекст

Утверждённый пользовательский дизайн находится в
`docs/superpowers/specs/2026-07-24-ai-analyst-discovery-skill-design.md`.
Существующий `sdd-process-companion` уже является управляемой входной точкой
GigaCode и устанавливается вместе с process package. Создание второго
конкурирующего skill увеличило бы неоднозначность выбора и сопровождения.

## Решение

Расширить один существующий companion двумя последовательными режимами:

1. `analyst-discovery` превращает сырую идею в подтверждённую сводку и
   source-aware черновики.
2. `guided-change` показывает и по отдельному разрешению выполняет допустимые
   действия существующего управляемого процесса.

Состояние интервью остаётся в текущем диалоге. Новый runtime и персистентная
машина состояний не создаются. Детерминированный CLI остаётся control plane,
а companion — объясняющим и черновым интерфейсом.

## Контракт разрешений

Разрешение на интервью, подтверждение итоговой сводки, разрешение на создание
черновиков, переход к первой команде и выполнение каждой команды являются
разными действиями. Ни одно из них не подразумевает следующее.

AI не создаёт trusted evidence из обычного ответа чата и не принимает решения
за Analyst, Tech Lead, Developer, QA либо владельца release/archive.

## Достоверность

Во время discovery companion поддерживает четыре класса утверждений:
`confirmed`, `proposed`, `unknown`, `conflict`. Итоговые документы остаются
человекочитаемыми; статусы показываются в сопроводительной evidence-сводке.

## Проверка

Автоматические tests проверяют наличие и порядок обязательных инструкций,
сохранение границ полномочий, установку skill из package и FAQ navigation.
Отдельный ручной walkthrough проверяет реальный диалог с простой фразы до
остановки перед первой командой.
